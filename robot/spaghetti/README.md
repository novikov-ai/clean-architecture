# Рефлексия

## Плюсы

По-максимуму использовал строгие типы: позиция, команда, состояние, что помогает избегать ошибок и делает код более читаемым.

## Минусы

Из явных минусов возникла проблема с командой "move" из-за углов. Не догадался использовать тригонометрические функции, а пошел в какой-то примитивный стиль. 

Есть проблема с оператором "Do", который представляет собой набор кейсов и в серьезной системе такой подход невозможно поддерживать.
