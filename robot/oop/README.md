# Рефлексия

## Плюсы

Логика управления роботом, а также парсинг входящих команд полностью инкапсулированы внутри структуры (класса).

Теперь мы можем абстрагироваться от конкретной реализации через АТД, описав спецификацию и завязавшись непосредственно в коде на интерфейс, что позволит "на ходу" подменять реализацию наших роботов.

## Минусы

Логика парсинга команд сейчас "вшита" в нашу реализацию робота, то есть для любого другого робота нам также потребуется ее написать вновь или вынести в отдельный пакет (библиотеку), что, конечно, предпочтительнее.  

Остается проблема с хранением состояния, которое будет обнуляться при каждом перезапуске.
